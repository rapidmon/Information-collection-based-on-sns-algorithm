- **트위터 수집기** — `screen_name` 없을 때 `i/web/status/{id}` URL 폴백

## 핵심 결정사항

| 결정 | 이유 |
|------|------|
| Posts, 수집 이력, 카테고리 → 로컬 SQLite/메모리 | Firebase 비용 절감, 불필요한 클라우드 의존 제거 |
| Briefings만 Firebase 유지 | 웹 대시보드(GitHub Pages)에서 직접 읽어야 하므로 |
| Cloudflare named tunnel (`api.cnvjb.uk`) | URL 고정, 재시작해도 config.js 수정 불필요 |
| 플립 카드 제거 | 키워드 chip이 요약을 대체하므로 뒤집을 이유 없음 |
| 트위터 URL fallback → `i/web/status/{id}` | screen_name 없는 기존 데이터도 링크 작동 |

## 주의사항

- **Cloudflare Tunnel 유지** — `cloudflared tunnel run --url http://localhost:8000 sns-briefing` 항상 켜두어야 GitHub Pages에서 데이터 로드 가능
- **Chrome 디버그 모드** — `--remote-debugging-port=9222` 없으면 수집 전부 실패
- **서버 재시작 시** — 터미널 창 유지해야 수집 계속됨 (Win+Shift+방향키로 이동해도 무관)
- **`data/posts.db`** — git에 포함되지 않음, 로컬에서만 존재
- **`.env`** — git에 포함되지 않음, 직접 관리 필요
- **AI 처리 `min_posts_to_process: 10`** — 미처리 게시물이 10건 미만이면 스킵됨
