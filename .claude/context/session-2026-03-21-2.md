## 핵심 결정사항

| 결정 | 이유 |
|------|------|
| `min_importance_for_briefing: 0.7` | 0.4 기준 시 하루 1,300건 → 113 토픽으로 너무 많음. 0.7로 340건 → 40 토픽이 적당 |
| `max_items: 0` (제한 없음) | 중요한 뉴스가 잘리지 않도록 AI 판단에 맡김 |
| `get_unbriefed limit: 10000` | 하루 수집량(~1,500건) 전체를 커버하기 위해 충분히 크게 설정 |
| 브리핑을 Firestore에 저장 | SQLite에는 `briefings` 테이블 없음, 기존 Firebase 인프라 활용 |
| `briefed_at` 마킹 방식 | 재브리핑 방지 — 한 번 브리핑된 게시물은 다음 날 브리핑에서 제외 |
