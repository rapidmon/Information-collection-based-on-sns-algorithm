## 핵심 결정사항

### 1. 색상 전략
- **파란색 accent** (#0057FF): 수치, 버튼, 강조 용도
- **회색 secondary** (#A8A8A6): 읽어야 하는 텍스트 (링크 기본값)
- **호버 시만 파란색**: 텍스트 링크는 기본 회색 → 호버 시 파란색

### 2. 애니메이션 타이밍
- 카드 호버: cubic-bezier(0.34, 1.56, 0.64, 1) - overshoot 느낌
- stat-number: 3초 펄스
- scanline: 8초 반복
- dot glow-ring: 3초 확산

### 3. 글래스모피즘 강도
- 카드: blur(20px) + rgba(26,26,24,0.5)
- 네비게이션: blur(20px) + linear-gradient 투명도
- 입력: blur(10px)
- 배지: blur(10px)

## 주의사항

### CSS 변수 사용
- 모든 색상은 `:root`에 정의된 변수 사용
- `--accent` (#0057FF): 강조 용도만
- `--text-secondary` (#A8A8A6): 링크 기본색

### 브리핑 콘텐츠 (briefing-content)
- `.briefing-content > div`: 배경/테두리 제거 (transparent 처리)
- `h1:first-of-type, h2:first-of-type`: 반복된 제목 숨김 (display: none)
- 이 규칙들은 LLM 생성 HTML 박스/제목 제거에 필수

### 의존성 설치 기록
```bash
pip install pyyaml pydantic-settings aiosmtplib playwright -q
```
- 모두 requirements.txt에 포함되어야 함

### 네비게이션 스크롤 애니메이션
- 스크롤 Y > 8px일 때 배경 어두워짐
- `nav.style.background` 인라인 변경 (CSS 대신)
- 성능상 이슈 없으나 필요시 CSS로 전환 가능
