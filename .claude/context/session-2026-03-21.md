### UTC → KST 타임스탬프 표시 버그 수정 (`a794247`)
- **문제**: SQLite에 UTC로 저장된 `collected_at` 문자열(`2026-03-20 15:48:58`)을 JavaScript `new Date()`로 파싱 시, `Z`(UTC 표시) 없으면 로컬 시간으로 오해석
- **증상**: `15:48 UTC` → 브라우저에서 `15:48 KST`로 표시 (실제로는 `00:48 KST`여야 함)
- **수정 위치**: `docs/js/app.js` `formatTimestamp()` 함수
- **수정 내용**: 타임존 표시 없는 문자열에 `'Z'` 추가 후 파싱

  ```javascript
  let dateInput = ts;
  if (typeof ts === 'string' && !ts.endsWith('Z') && !ts.includes('+')) {
      dateInput = ts.replace(' ', 'T') + 'Z';
  }
  const date = ts.toDate ? ts.toDate() : new Date(dateInput);
  ```
